accessibility on our page

-everything to help the user-

semantic html structure

On our main page we have used semantics tags like:
<header>,<nav>,<main>,<section> and <footer>
to give our document structer, and help screen readers to navigate to diffrent desired parts.

alternative text for images

We provide "alt" text for pictures, explaining their contents in case of pictures not showing or user is disabeled.

keyboard navigation

If for some reason the user cannot see or use a mouse there is a possibility of using only tab to navigate to interactive elements like:
Links, buttons, forms etc

clear focus indicators

Visibal focus indicators to help users know their location on the website.

aria(accessible rich internet applications) landmarks and roles

Implemented ARIA attributes to add extra information for assistive technologies(e.g screen readers) to understand dynamic content.
This includes aria-label, aria-expanded, aria-live and adding roles like role="button".

text and content contrast

Using colors that maintain a high contrast ratio between text and background. Help users read otherwise hard to read text.

responsive and scalable text

making sure the layout stays intact while zooming to (e.g 200%) and ensuring it still looks good. This also includes making the site work on diffrent devices with diffrent display sizes.

descriptive links

Links have text that describe where it will take you instead of being a link without context.

forms and labels

ensured every input element have a label element with descriptive text to explain its purpose.

skip navigation links

"skip to content" links that takes you to diffrent parts of the document without repetetive navigation.

multimedia accessibility

used captions, transcripts and audio descriptions for audio and video where it was possible. aria-live is a good exampel for dynamic content.

accessible language and readability

clear and concise language at a readable level to make it easier to understand. We tried to avoid jargon and made plain and simple explanations for "complex information"

consistent layout

tried to have a predictable layout that doesnt change across the page. e.g consistent navigation elements and structure.
